# 提示词框架

优化时先按任务类型挑一个框架当**骨架**，再用 `principles.md` 的维度把它填满。框架是骨架不是教条——能拼、能删，目标是把提示词组织清楚。

## 选哪个（快速启发式）

| 任务类型 | 用 | 一句话 |
|---|---|---|
| 写作 / 内容 / 文案 / 邮件 | **CO-STAR** | 成败常在"受众 + 语气" |
| 简单、一句话能说清、要得快 | **RTF** | 三件套，够用就好 |
| 多步骤 / 调研 / 分析 / 代码 | **RISEN** | 强调步骤与边界 |
| 系统提示词 / 角色 / 智能体 | **LangGPT 角色模板** | 给 AI 定一个稳定人格 |
| 拿不准 | RTF 起步 | 缺什么再从别的框架补 |

## CO-STAR（写作 / 内容）
- **C**ontext 背景
- **O**bjective 目标（要它做什么）
- **S**tyle 文体风格
- **T**one 语气
- **A**udience 受众
- **R**esponse 输出格式

适合文案、邮件、文章、营销。受众和语气没说清，输出就会跑偏。

## RTF（简单快速）
- **R**ole 角色
- **T**ask 任务
- **F**ormat 输出格式

最轻量。一句话能讲清的任务用它，别给简单任务套复杂框架。

## RISEN（多步骤 / 流程 / 调研）
- **R**ole 角色
- **I**nstructions 任务说明
- **S**teps 步骤（按序执行）
- **E**nd goal 最终目标 / 成功标准
- **N**arrowing 收窄约束（范围、禁止项）

适合深度调研、分析报告、代码任务——有明确执行顺序时最强。

## LangGPT 角色模板（系统提示词 / 智能体）
给一个要**反复使用、行为要稳定**的 AI 角色定人格时用。模板是骨架不是教条——用不到的小节（如 Skills、Workflow）该删就删，别为凑结构硬塞：

```
# Role: <角色名>

## Profile
- description: <一句话定位>

## Skills
- <能力1>
- <能力2>

## Rules
- <必须遵守的>
- <绝对禁止的>

## Workflow
1. <第一步>
2. <第二步>

## Initialization
作为 <Role>，严格遵守 Rules，用 <语言> 与用户交流，开场先 <做什么>。
```

适合客服 bot、专家助手、智能体等系统提示词——这正是一次性"用户提示词"覆盖不到的场景。

> 来源：CO-STAR / RTF / RISEN 是业界通用提示词框架；角色模板源自 LangGPT（结构化提示词范式）。
