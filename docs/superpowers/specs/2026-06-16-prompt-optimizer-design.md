# 提示词优化助手（prompt-optimizer）设计文档

- 日期：2026-06-16
- 形态：Claude Code **项目级 skill**（仅在本项目目录下生效，不进全局 `~/.claude/skills/`）
- 目标：在 Claude Code（或类似工具）中被唤起，把用户的草稿提示词或一句话需求，优化成结构化、高质量的提示词，供用户复制到各种 AI（ChatGPT / 豆包 / Claude 等）中使用。

---

## 1. 目录结构

```
<项目根>/
└── .claude/
    └── skills/
        └── prompt-optimizer/
            ├── SKILL.md              # 主文件：触发条件 + 优化逻辑 + 输出格式
            └── references/
                ├── principles.md     # 通用提示词工程原则
                ├── ai-profiles.md    # 各 AI 特点与适配要点
                └── scenarios.md      # 各场景结构模板
```

主文件保持精简，知识库（各 AI 要点、各场景模板）拆到 `references/`，便于扩展：新增 AI 或场景只改对应参考文档，不动主逻辑。

---

## 2. 触发方式

通过 `SKILL.md` 的 `description` 配置触发词，用户用自然语言即可唤起。覆盖的表达包括：
优化提示词、提示词优化、帮我写提示词、润色提示词、帮我写个 prompt、prompt 优化、把这段需求变成提示词 等。

---

## 3. 核心工作流

```
① 接收输入（草稿提示词 或 一句话需求）
        ↓
② 判断信息是否充分？
   检查 4 个关键点：任务目标 / 受众或用途 / 输出格式 / 关键约束
   ├─ 缺 2 个以上 → 先反问（一次性问完，尽量给选项，简洁）
   └─ 基本齐全   → 直接优化
        ↓
③ 识别可选参数：目标 AI + 场景
   ├─ 用户明说（如"给豆包写个深度调研的"）→ 加载对应适配
   └─ 未说          → 走通用版
        ↓
④ 套用优化方法（通用原则 + 场景模板 + AI 适配）
        ↓
⑤ 输出：代码块里的最终提示词 + 下方简短"改动说明"
```

第②步即"智能判断"：需求清楚直接改写，信息不足才反问。

**反问规则**：最多问一轮，集中一次性问完，尽量用多选项形式，避免来回拉扯。

---

## 4. 优化方法（principles.md 的维度）

从以下维度把粗糙输入补成结构化好提示词：

- **角色/视角**：让 AI 站在合适的专家身份
- **背景/上下文**：补全 AI 需要但不知道的信息
- **明确任务**：把模糊诉求变成具体、可执行的指令
- **约束与要求**：范围、长度、语气、必须/禁止项
- **输出格式**：表格/列表/分段/JSON 等
- **示例或推理引导**：必要时给 few-shot 或"分步思考"
- **去歧义、补隐含需求**：让 AI 更懂用户真实意图

---

## 5. 参数分发机制

自然语言识别，无需记命令：

| 用户表达 | 行为 |
|---|---|
| "帮我优化这个提示词：……" | 通用优化 |
| "给 ChatGPT 写个提示词……" | 加载 ChatGPT 适配 |
| "我要个深度调研的提示词……" | 套用深度调研模板 |
| "给豆包做深度调研用的……" | AI + 场景 双适配 |

**初始内置场景**（可扩展）：一般搜索问答、深度调研、写作/文案、代码/技术、角色扮演、数据提取/结构化输出。

**初始内置 AI**（可扩展）：ChatGPT、豆包、Claude、通用（默认）。

默认走通用版；仅当用户明确指定时才加载对应 AI/场景适配，避免增加交互摩擦。

---

## 6. 输出格式

固定格式，便于一键复制：

````
（优化后的最终提示词，独立代码块，拿走即用）
````

**改动要点**（3-5 条，改了什么 + 为什么）
> 适配说明：若用了某 AI/场景适配，附一句

---

## 7. 异常处理

- **输入太模糊**（连问都难）→ 先问"你想让 AI 帮你做什么"
- **提示词很长**→ 保留原意做结构化重排，说明主要改动
- **想再改**→ 基于上一版迭代
- **语言**→ 默认跟随用户输入语言（中文）

---

## 8. 验证方式

实现后用 3 个用例自测：
1. **模糊输入**（如"帮我写个提示词"）→ 应触发反问
2. **清晰输入**（含目标/格式/约束）→ 应直接改写并给出代码块
3. **带 AI + 场景**（如"给豆包写深度调研提示词"）→ 应正确加载对应适配并体现在结果中
